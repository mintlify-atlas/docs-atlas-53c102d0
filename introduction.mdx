ML Intern is an open-source CLI agent that autonomously researches, writes, and ships machine learning code using the Hugging Face ecosystem. Instead of switching between paper search, dataset exploration, training scripts, and cloud compute dashboards, you describe what you want in plain language and ML Intern handles the rest — from reading ArXiv papers to pushing trained models to the Hub.

## How it works

ML Intern runs an agentic loop that repeatedly calls an LLM (Claude by default, via Anthropic or AWS Bedrock), parses the tool calls it returns, and executes them through a `ToolRouter`. Each iteration follows the same pattern:

1. The LLM receives your message plus the full conversation history and a set of tool specs.
2. If the LLM returns tool calls, ML Intern checks whether any require your approval (cloud jobs, destructive repo operations, sandbox commands).
3. Approved calls are executed by the `ToolRouter`, and their results are added back to the conversation context.
4. The loop continues until the LLM returns a response with no tool calls, signalling it has finished the task.

The loop runs for up to 300 iterations by default (configurable). A built-in doom-loop detector monitors for repeated tool patterns and injects corrective prompts if the agent gets stuck.

### Tool categories

The `ToolRouter` exposes tools across several categories:

| Category | What it covers |
|---|---|
| HF docs & research | ArXiv paper search, HF documentation lookup, GitHub code search |
| HF ecosystem | Model/dataset repos, Hub jobs, repo files, Git operations |
| Sandbox & local | Local code execution, file operations, reliability checks |
| Planning | Structured task planning and sub-task tracking |
| MCP servers | Any tools exposed by MCP servers you configure |

### Context management

ML Intern tracks conversation history in a `ContextManager`. When the context grows beyond 170k tokens, it automatically compacts — summarising older turns — so long tasks don't hit model limits. Sessions can be saved to a Hugging Face dataset repo after each turn, giving you a persistent audit trail.

<Note>
  ML Intern requires an HF token and an LLM API key (Anthropic or a configured Bedrock profile). If no `HF_TOKEN` is set, the CLI will prompt you to paste one on first launch.
</Note>
