---
title: "Get started with ML Intern: install and run first task"
description: "Install ML Intern from source using uv, set your API credentials, and run your first autonomous ML task in interactive or headless mode."
---

ML Intern is installed from source and runs as a global CLI command. You need Python 3.11 or later, [uv](https://docs.astral.sh/uv/), and API credentials for Hugging Face plus at least one LLM provider. The steps below take you from a fresh machine to running your first task.

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the ML Intern repo and enter the project directory:

    ```bash
    git clone git@github.com:huggingface/ml-intern.git
    cd ml-intern
    ```
  </Step>
  <Step title="Install dependencies">
    Use `uv` to install all dependencies into a managed virtual environment:

    ```bash
    uv sync
    ```
  </Step>
  <Step title="Install the CLI globally">
    Install ML Intern as a global tool so the `ml-intern` command works from any directory:

    ```bash
    uv tool install -e .
    ```
  </Step>
  <Step title="Set your credentials">
    Create a `.env` file in the project root with your API keys:

    ```bash .env
    HF_TOKEN=<your-hugging-face-token>
    ANTHROPIC_API_KEY=<your-anthropic-api-key>
    GITHUB_TOKEN=<your-github-personal-access-token>
    ```

    | Variable | Required | Description |
    |---|---|---|
    | `HF_TOKEN` | Yes | Hugging Face access token. Used to read and write repos, datasets, and jobs on the Hub. If not set, the CLI prompts you on first launch. |
    | `ANTHROPIC_API_KEY` | Yes (if using Anthropic) | API key for Claude models. Required when `model_name` points to an Anthropic model. |
    | `GITHUB_TOKEN` | Recommended | Personal access token for GitHub code search. Without it, GitHub-related tools are unavailable. |

    <Tip>
      Get your HF token at [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens). To create a GitHub token, follow the [GitHub fine-grained PAT guide](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens#creating-a-fine-grained-personal-access-token).
    </Tip>
  </Step>
</Steps>

## Run your first task

### Interactive mode

Launch an interactive chat session with:

```bash
ml-intern
```

The agent starts, loads its tools, and waits for your input. Type a task in plain language:

```
> fine-tune a small vision model on the Oxford Pets dataset and push it to the Hub
```

ML Intern will research the task, plan the steps, write training code, and ask for your approval before dispatching any cloud jobs or making changes to repositories.

### Headless mode

Pass a prompt directly to run a single task and exit:

```bash
ml-intern "find bird classification datasets on the Hub and summarise the three largest ones"
```

In headless mode, all tool calls are auto-approved (equivalent to `yolo_mode = true`). Use this for scripting and automation.

## CLI flags

| Flag | Description |
|---|---|
| `--model <id>` | Override the model set in `configs/main_agent_config.json`. Accepts any LiteLLM-compatible model ID (e.g. `anthropic/claude-opus-4-6`). |
| `--max-iterations <n>` | Maximum number of LLM calls per agent turn. Defaults to the value in config (300). Pass `-1` for effectively unlimited. |
| `--no-stream` | Disable token streaming and use non-streaming LLM calls instead. Useful when the model or provider doesn't support streaming. |

```bash
# Use a specific model for this run
ml-intern --model anthropic/claude-opus-4-6 "your prompt"

# Cap iterations to avoid runaway tasks
ml-intern --max-iterations 50 "your prompt"

# Disable streaming
ml-intern --no-stream "your prompt"
```

## Slash commands

Inside an interactive session, you can control the agent with slash commands:

| Command | Description |
|---|---|
| `/help` | Show all available commands. |
| `/model [id]` | List available models, or switch to a specific model. |
| `/effort [level]` | Set the reasoning effort level: `minimal`, `low`, `medium`, `high`, `xhigh`, `max`, or `off`. |
| `/yolo` | Toggle YOLO mode — auto-approve all tool calls without confirmation. |
| `/compact` | Manually compact the conversation context. |
| `/undo` | Undo the last agent turn. |
| `/status` | Show current model, reasoning effort, turn count, and context size. |
